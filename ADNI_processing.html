ADNI Scan Download 


Step 1: Record date of download in /home/jagust/susanl/scripts/adni/adni_notes.txt
Step 2: Make a new directory for downloads in /home/jagust/susanl/adni/download_from_loni using date and image description information (eg: AV45_01.13.14)
Step 3: Download scans from http://adni.loni.usc.edu

  • Sign in using the login dovairre@gmail.com; password: do_over!
  • Click on the Download tab then "Image Collections" then "Advanced Search (beta)"
  • Choose the correct project and modality, and in the image description box, copy the following phrases for the type of scan you are trying to download:
    • FDGPET: "Coreg, Avg, Std Img and Vox Siz, Uniform Resolution"
    • PIB: PIB Coreg, Avg, Std Img and Vox Siz, Uniform Resolution
    • AV45: AV45 Coreg, Avg, Std Img and Vox Siz, Uniform Resolution
    • MRI: *N3;* or for ADNI2: *N3m*

  • For Florbetapir scans, some were downloaded on a different account, so you also have to specify the dates of scans you are searching for. To do so check the log from Step 1 to see when the last download occured, then take scans from the previous study date through today 
  • Select all results in the upper right corner and click add to the appropraite collection (for AV45 scans make a new collection that specifies the scan dates) 
  • Choose the "Not Downloaded" scans, select All, click on "NiFTI" in the upper right corner then click "Download" - when prompted for a download folder choose your newly created folder in /home/jagust/susanl/adni/download_from_loni 
  • Repeat for each modality you want to download 

Step 4: Organize the scans onto ARDA by running the script:/home/jagust/susanl/scripts/adni/organize_data.py and selecting the "/ADNI" directory within downloaded data directory 

Step 5: Delete downloaded scans on raid once they have been organized onto arda to save space

FDG Processing 

Step 1: After organizing scans onto arda, spatially normalize them by running ~susanl/scripts/adni/spatnorm_PET2PET_adni_10.23.12.py
Step 2: Take a look at pre and post spatially normalized FDG images using spm in matlab-spm5
Step 3: Extract data using ~/scripts/adni/extract_rois_pet2pet_susan_adni_10.23.12.py 

  • usage: python ~/scripts/adni/extract_rois_pet2pet_susan_adni_10.23.12.py Normal
  • Output goes into "/home/jagust/susanl/adni/extracted_data/" 

Step 4: Sort output file by PTID then Date and delete unnecessary columns 

Step 5: If you want to, find / replace new VISCODE_NUM variable:Find/replace new VISCODE_NUM variable: 1=Baseline,ADNI2 Initial Visit 2=Month6 3=Month12 4=Month18 5=Month24 6=Month36 7=Month48 8=Month60 9=ADNI2 Initial Visit-Cont Pt 

Step 6: Save the text output as a csv file. The output is broken down into 5 ROIs at each visit, and the value we want is the average of these 5 subregions. You can use any method to average each subject's 5 subregions at each visit. Mine was as follows:

  • Change the PTID into RID by only keeping the last four numbers (replace *S_ with blanks) and rename the column "RID" 
  • Sort by RID and then by Date so that the each subregion per visit is in 5 consecutive rows and the output dates are in the correct order
  • Put the date in the format 'yyyy-mm-dd' 
  • Run it through the FDGmeans_and_examdates script in ~/afero/matlab. Change the name and path of the input and output file before running. It has to be comma delimited, not tab delimited.

(Use updated scripts in ~ahorng/matlab/)

Step 1: Using Matlab-spm5 run the "Step 1_FindFlorbetapir2Analyze_3Visits.m" Matlab processing script in ~/afero/matlab to identify scans that still need an MRI to be processed 

  • This script outputs a list of scans whose MRIs have yet to be processed to freesurfer

  • Run Freesurfer analysis on these subjects (See Freesurfer Processing)
  • After MRIs have been successfully processed, run the Step 1 script again to see which scans still do not have MRIs and record it in the adni notes text folder (/home/jagust/adni/adni_av45/adni_notes.txt)


Step 2: Use Matlab-spm5 to run the Step 2 script "Step2_CoregPET2nu_3Visits.m" in ~/afero/matlab to coregister the AV45 to MRI space 

  • Note: This script takes over the Unix command line (and/or whatever else you might be working on)... best to run overnight or while not using NX actively


Step 3: Run the Step 3 script "Step3_CheckCoreg_MoveFiles_3Visits.m" in ~/afero/matlab to check coregistratino visually. Script will prompt you to inspect the coregistration and respond if it is okay
Step 4: Run the Step 4 script "Step4_Create_whitemask70aparc_nuspace_1113_AF.m" to create the eroded white matter mask
Step 5: Run the Step 5 script "Step5_ExtractMeansStdevSizes_3Visits.m" in ~/afero/matlab. This script calls on another called "FextractROIvalues_wWhitemask.m", which identifies and extracts the means of the freesurfer subregions. I have many versions of this file that I edited to do different types of extraction, but this is the default and should be called upon normally. This script creates the final output file in the folder "~/susanl/adni_av45_av45_(scannumber)". Open the file in NX by using oocalc, or transfer it to your local machine and use excel.
Excel Header Output Key
Step 6: For each person at each timepoint the composite region must be normalized by whatever reference region you are using (wcereb, gcereb, wm70, etc). This is done in excel by simply dividing the composite by the ref region. Use the normalized ref regions to find the difference and percent change per year in uptake between visit 1 and visit 2. 

  • The "big ref" region is created by averaging WM70, brainstem, and whole cerebellum 
  • The "WMRATIO" region is more complicated. For details on the theory talk to Susan. But in practice use the whole cerebellum reference region for Visit 1. Then for Visit 2, take the whole cerebellum normalized Visit 2 values and multiply by the ratio of WM70 Visit 1 over WM70 Visit 2 (Hence why we call it WM ratio). The WM ratio normalization is summarized in the following equations, where CtxV1=CtxV1/WhCerebV1, CtxV2=CtxV2/WhCerebV2, WMV1=WMV1/WhCerebV1, and WMV2=WMV2/WhCerebV2: $$\text{SUVR V2} = \frac{\text{CtxV2}_\text{{WCV2}}}{\frac{\text{WM}_\text{{V2}}}{\text{WM}_\text{{V1}}}}$$


Step 7: Scan dates come from the "META" files for each modality. This file includes all scans, including unprocessed ones. We are interested in in the scans whose "SEQUENCE" column matches the "Image Descriptions" above. Each Subject's "RID" is the final four digits of the Subject number.

Data Extraction and Compilation 

Data is compiled into the spreadsheet "FDG_AV45_COGdata.xlsx" saved in the shared ADNI folder. The data is pulled from many sources. The AV45 information comes from our processing of the downloaded scans and is output in Step5 of Processing Florbetapir Scans and the FDG information is extracted from FDG Processing. However, most information is downloaded from http://adni.loni.usc.edu . 

  • To download csv data, sign in again and click on "Dowload" followed by "Study Data"
  • Tests that we extract: MMSE, AVlT, ADAS-cog 
    • AVLT has 5 free recall trials that need to be added together. The columns are called AVTOT1,2,3,4,5. We don't need AVERR scores. The sum must be calculated for each of these 5 columns. However, some cells have negative numbers in them meaning the trial was invalid, so be careful not to add negative and positive numbers together. Some subjects will have a recall score of 0, which is a valid result and should be added. 
    • ADAS-cog has two spreadsheets. The ADNI1 spreadsheet is called ADASSCORES.csv and the column of interest is TOTAL11. The ADNIGO/2 spreadsheet column of interest is called TOTSCORE. Unfortunately, one of the spreadsheets has scores to 2 significant digits, and the other has 0 significant digits, but there's nothing we can do about it, so we have to round everything to 0 significant digits.
  • Exam dates come from the Registry CSV file and must be added to the cognitive spreadsheets by matching RID and VISCODE2 numbers. (I do this by combining the RID and VISCODE2 columns in excel for both spreadsheets, then using vlookup) 
  • Depending on your goal, you may be trying to find scores corresponding to AV45 scans, or you may be interested in looking at change over all socres. If you need all scores, you will have to convert the spreadsheet from long format to wide format. There are various techniques for this including writing your own script or using SPSS's built in function. 
  • If you are trying to find a single score closest to a florbetapir date, you can keep the excel file in long format. Add the date of the florbetapir scan using vlookup for every row. Then make a new column with the date difference between the cognitive test date and the florbetapir date. Sort the spreadsheet by date difference, and only take the tests that are within 3 months of the florbetapir scan.

Freesurfer Processing of MRI scans 

MRIs must be processed for AV45 scans who do not yet have one. Make sure that you have created a file with subjects who need MRIs processed (See Processing Florbetapir Scans)
Step 1: Create temp freesurfer directory to put output in (in ~/susanl/adni_av45) and update freesurfer directory in your ~/.bashrc file
Step 2: Edit autorecon* scripts in ~/susanl/scripts/adni 

  • There are two autorecon scripts (06/13 versions). One is a print only file to show which subjects don't have MRIs. The other is the actual processing script. Both need to be edited in a few places: the subjid input in line 14 and the FSDIR in line 69. In the print only file the outfile (line 9) must also be changed
  • Run the print only file first to make sure it runs smoothly and to see which subjects do not have MRIs 

Step 3: Submit the actual processing script to the cluster. To do this we use the "submit-python*.sh" shell script. This script needs to be edited to reflect the number of subjects you are running (line 10) and which script is being submitted (line 47)

  • To submit to the cluster: type qsub submit-python7.sh (or whatever version you are submitting) 
  • To check the status of the submission type qstat 
  • Freesurfer output will go into the freesurfer folder you specified. Check error logs to see if the subject exited with errors. If so, try running a different MRI or using suggested tags 

Step 4: Record problems / subjects missing MRIs in ~/susanl/scripts/adni/adni_notes.txt 

If you're getting freesurfer errors, a good fix is updating freesurfer to the newest version: change FREESURFER_HOME in ~/.bashrc, then re-login

Voxelwise Comparisons 

To do a voxelwise analysis, create a spreadsheet with the full path to the input scans in the first column, then covariates in the following column
To run the analysis: 

  • Open matlab then type "spm" to open "spm", then click on "PET/SPECT" or "PET&VBM" depending on the version
  • Click on "Basic Models" in the spm window
  • Input the output directory under "directory"
  • For design, usually we are doing a 2-sample t test comparing uptake in two different groups. Choose this option under "design"
  • Choose the scans you want to analyze by clicking "Scans" then "Select files." This brings up a gui where you can select them, but it is easiest to click the small "Ed" button. With this button you can copy your list of scans directly from your spreadsheet. Pay attention to order because scan order must match covariate order
  • Choose the number of covariates. Input them in the same order as the scans (group 1 first followed by group 2)
  • To use a grey matter mask, click on "Explicit Mask" and choose "/home/jagust/susanl/templates/grey_matter_mask_from_beth.nii"
  • Choose your output folder
  • Save your model if you want then click "run", this produces a graphical representation of the stat model
  • This will create a file called "SPM.mat" in the output directory. Therefore every test requires its own directory.
  • Click "estimate" then "results" in the spm window (choosing the correct SPM.mat file). Click "Define new contrast" and choose values for your variables of interest. If it is a paired t test it should be 1 and -1 for your two varialbes and 0s for the covariates 
  • To view the results use mricron and open the overlay "spmT*.hdr"
